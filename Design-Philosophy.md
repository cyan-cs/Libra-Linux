# Libra Linux - 設計について

## 1. Libra Linux について

**Libra Linux** は **Arch Linux** を基盤とした Linux ディストリビューションである。  
軽量性と美しさ、そして日常で使える実用性のバランスを意識して設計している。

主な対象は開発者や技術者だが、  
いつもとは少し違う、プロ向けに近い環境を使ってみたい一般ユーザーも想定している。

Libra Linux を設計する上で、  
判断の基準としている考え方は次の一つだけである。

> **極端に振れないこと**

軽さだけを優先しない。  
便利さだけにも寄らない。  
安全性だけを絶対視することもしない。

その都度、何を優先するかを **天秤（Libra）にかけて決める**、  
という立ち位置を取っている。

---

## 2. 中核となる考え方

### バランスを重視する

Libra Linux では、次の三つを同時に考慮する。

- **性能（Performance）**
- **安全性（Security）**
- **実用性（Usability）**

いずれか一つだけを理由に、他を切り捨てる設計は基本的に行わない。

---

### Arch Linux との関係

Libra Linux は Arch Linux のフォークではない。  
Arch Linux を基盤とした、比較的小さな派生という位置付けである。

以下の点は変えない。

- pacman を使う
- systemd を使う
- Arch Linux の設計思想を尊重する

一方で、次のようなことは意図的に行わない。

- 内部構造が見えなくなる変更
- 理由の分からない自動化
- 必要性の説明できない改変

Arch Linux を理解している人であれば、  
Libra Linux も特別な説明なしに扱える状態を目指している。

---

## 3. 想定している利用者

Libra Linux は、次のような人が使うことを想定している。

- 開発者
- システムエンジニア
- OS やカーネルを学びたい人
- セキュリティを意識して環境を構築したい人
- OS の挙動を把握しながら使いたい一般ユーザー

特定の層に強く最適化することはしていない。

---

## 4. セキュリティに対する考え方

Libra Linux では、次の前提を置いている。

- ユーザーは判断力を持っている
- ただし、人間はミスをする
- OS は、そのミスを前提に振る舞うべきである

---

### 初期状態での方針

初期設定では、次のような状態になっている。

- ファイアウォールは有効
- root の SSH ログインは禁止  
  （必要であれば、明示的に許可できる）

設定を緩めることは可能だが、  
最初から緩めた状態にはしない。

これは制限というより、  
「どこまで許可するかを自分で決めるための出発点」と考えている。

---

## 5. プロジェクトとしての姿勢

Libra Linux では、次のような点を重視している。

- 便利さよりも、仕組みが見えること
- 全自動よりも、意図が分かる操作
- 多機能さよりも、構造の単純さ
- 過剰でない現実的な安全性

---

## 6. 目指さないこと

Libra Linux は、次のようなものを目標にはしていない。

- Arch Linux の代替になること
- 複雑な処理を裏で隠すこと
- すべてを自動で設定すること
- GUI を前提とした設計

---

## 7. まとめ

Libra Linux は、  
数値や派手さを競う OS ではない。

構成を把握でき、  
自分で管理できる状態を維持しながら、  
長く使える環境であることを目指している。

> 自分で理解できる範囲にあるシステムの方が、扱いやすく、安全である。

---

## 8. Fork について

Libra Linux では、Fork を制限しない。

用途や考え方が異なれば、  
派生が生まれるのは自然なことだと考えている。

Libra Linux 本体は、次の点を前提として開発を続ける。

- Arch Linux との互換性を保つこと
- 変更点を説明できること
- 特定用途に寄りすぎないこと

Fork の目的や方向性については、  
それぞれのプロジェクトが判断するものとする。

Libra Linux は完成形ではなく、  
一つの土台として存在することを選んでいる。
